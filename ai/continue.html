Continue on FreeAppStore

Build a free app using Continue. The agent reads the platform's SKILLS.md and knows the brand rules, the CLI, and the deploy flow before it writes a line of code.

Before you start: install Node 22+ and the FreeAppStore CLI.
npm i -g @freeappstore/cli fas login

Set up Continue

  1. Install the Continue extension

    VS Code: search "Continue" in Extensions and install. JetBrains: install from the marketplace.

  2. Scaffold the app
    fas init my-app code my-app
  3. Add platform context to .continue/rules

    Continue reads custom rules from .continue/rules:

    mkdir -p .continue/rules && cat > .continue/rules/freeappstore.md <<'EOF' # Platform: FreeAppStore Always follow https://raw.githubusercontent.com/freeappstore-online/ops/main/SKILLS.md. Constraints: - Manrope + Fraunces fonts only. - Brand CSS tokens via var(--accent), var(--ink), var(--paper). - Zero tracking SDKs. - Use the Shell component. - Push to main = deploy. EOF
  4. Open Continue chat (Cmd-L in VS Code)

    Pick a model in the dropdown. Continue supports OpenAI, Anthropic, Mistral, Ollama (local), Together, etc.

Prompt the build

@codebase Build a unit-converter app: category picker (length, weight, temperature) on the left, input/output on the right. Persist last unit pair per category. Use the Shell layout.
@codebase Add fuzzy search across phrase target and meaning to my mandarin-english app. Highlight matches.

Iterate

  1. Run the dev server
    cd my-app && pnpm install && pnpm dev

    The agent will tell you which file it edited; refresh the browser to see the change.

  2. Run compliance checks
    fas check

    Catches missing brand fonts, tracking SDKs, oversized bundles, and missing PWA manifest before publish.

  3. Publish
    fas publish

    Provisions github.com/freeappstore-online/my-app, the Cloudflare Pages project, the DNS record, custom subdomain at my-app.freeappstore.online, and the storefront entry — atomically.

  4. Push your code
    git push upstream main

    Auto-deploys via Cloudflare Pages in ~30 seconds. Future commits to main deploy automatically.

What the AI knows (and doesn't ask you for)

Once SKILLS.md is loaded, the agent will:

  • Use the platform's CSS tokens (--accent, --ink, --paper, …) — never redefine them.
  • Use Manrope (body) + Fraunces (display) — never custom display fonts.
  • Refuse to add Google Analytics, Plausible, Mixpanel, or any tracker.
  • Use localStorage for persistence (or @freeappstore/sdk KV for connected apps).
  • Push to main for deploy. Never wrangler pages deploy manually.
  • Never ask you for Cloudflare or GitHub tokens.

The compliance suite (fas check) enforces all of the above on every push, so even if the AI slips, the platform catches it.

Get the source

The CLI, SDK, and compliance package live at https://github.com/freeappstore-online/sdk. Reading the source is allowed — and often illuminating when you're trying to coax a difficult prompt.

Tips for Continue specifically

  • @codebase in chat lets the agent search across the whole repo.
  • Cmd-I for inline edits in the current file.
  • Configure local models via Ollama for offline coding — Continue's strength.
  • The config.json in ~/.continue/ lets you fine-tune system prompts per model.
