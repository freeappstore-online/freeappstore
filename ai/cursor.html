Cursor on FreeAppStore

Build a free app using Cursor. The agent reads the platform's SKILLS.md and knows the brand rules, the CLI, and the deploy flow before it writes a line of code.

Before you start: install Node 22+ and the FreeAppStore CLI.
npm i -g @freeappstore/cli fas login

Set up Cursor

  1. Install Cursor

    Download from cursor.com and install. Sign in with your account.

  2. Scaffold the app
    fas init my-app cursor my-app

    The cursor CLI ships when you install Cursor (similar to code for VS Code).

  3. Add platform context to .cursorrules

    Cursor reads .cursorrules at the project root and prepends it to every prompt:

    cat > .cursorrules <<'EOF' # Platform: FreeAppStore Always read https://raw.githubusercontent.com/freeappstore-online/ops/main/SKILLS.md before suggesting changes. The platform enforces: - Manrope + Fraunces fonts only (no custom display fonts). - Brand CSS tokens via var(--accent) etc — don't redefine them. - Zero tracking SDKs. - Push to main = deploy. No staging. Use the Shell component for layout. Edit web/src/App.tsx for the UI. EOF
  4. Open the AI chat (Cmd-L)

    Start composing your prompt. The .cursorrules file is included automatically.

Prompt the build

Build a Pomodoro timer app: 25-minute work, 5-minute break, configurable. Audio chime on phase change (use Web Audio API, no audio files). Persist the current session in localStorage so a refresh doesn't reset.
Add tab navigation to my app: list view, stats view, settings. Use the URL hash so back/forward works. Keep the Shell layout intact.

Iterate

  1. Run the dev server
    cd my-app && pnpm install && pnpm dev

    The agent will tell you which file it edited; refresh the browser to see the change.

  2. Run compliance checks
    fas check

    Catches missing brand fonts, tracking SDKs, oversized bundles, and missing PWA manifest before publish.

  3. Publish
    fas publish

    Provisions github.com/freeappstore-online/my-app, the Cloudflare Pages project, the DNS record, custom subdomain at my-app.freeappstore.online, and the storefront entry — atomically.

  4. Push your code
    git push upstream main

    Auto-deploys via Cloudflare Pages in ~30 seconds. Future commits to main deploy automatically.

What the AI knows (and doesn't ask you for)

Once SKILLS.md is loaded, the agent will:

  • Use the platform's CSS tokens (--accent, --ink, --paper, …) — never redefine them.
  • Use Manrope (body) + Fraunces (display) — never custom display fonts.
  • Refuse to add Google Analytics, Plausible, Mixpanel, or any tracker.
  • Use localStorage for persistence (or @freeappstore/sdk KV for connected apps).
  • Push to main for deploy. Never wrangler pages deploy manually.
  • Never ask you for Cloudflare or GitHub tokens.

The compliance suite (fas check) enforces all of the above on every push, so even if the AI slips, the platform catches it.

Get the source

The CLI, SDK, and compliance package live at https://github.com/freeappstore-online/sdk. Reading the source is allowed — and often illuminating when you're trying to coax a difficult prompt.

Tips for Cursor specifically

  • Cmd-K for inline edits in the current file. Cmd-L for chat with project context.
  • Right-click a file → Add to context when you want the agent to see beyond what it auto-discovers.
  • Composer (Cmd-I) handles multi-file refactors better than chat.
  • Switch the model via the dropdown — Claude Sonnet handles the platform's TypeScript + React idioms well.
