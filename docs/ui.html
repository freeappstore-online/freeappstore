UI Component Library

Drop-in React components for FreeAppStore apps. Composable primitives, design tokens, and a zero-config shell.

On this page

Choose your level

Pick the abstraction that fits your needs:

Level 1: Full Shell

Zero-config. One component gives you auth, topbar, theme, footer.

<FasShell app={fas}>
  <MyApp />
</FasShell>

Level 2: Composable

Custom layout with SDK components.

import { Avatar, ProfileMenu,
  ThemeToggle } from
  '@freeappstore/sdk/ui'

Level 3: Hooks only

Full control. Build your own UI.

import { useAuth, useTheme }
  from '@freeappstore/sdk/hooks'

Level 4: Profile page

Dedicated settings page for any route.

<Route path="/profile"
  element={<ProfilePage
    app={fas} />} />

Design Tokens

SDK components reference CSS custom properties. Your app's stylesheet defines the values. If a variable isn't defined, components fall back to sensible defaults.

TokenLightDarkPurpose
--bg #f8fafc #0f172a Page background
--surface #ffffff #1e293b Card/panel background
--surface-2 #f1f5f9 #0f172a Secondary surface
--ink #1e293b #f1f5f9 Primary text
--ink-strong #0f172a #ffffff Emphasized text
--muted #64748b #94a3b8 Secondary text
--border #e2e8f0 #334155 Borders
--accent #2563eb #3b82f6 Primary action
--accent-hover #1d4ed8 #2563eb Action hover
--accent-soft #eff6ff #1e3a5f Accent background
--radius 0.75rem Default border radius
--radius-sm 0.5rem Small border radius
--shadow 0 1px 3px rgba(15,23,42,0.08) Card shadow

The SDK never injects CSS variables. Your app's stylesheet is the source of truth. The template-standalone ships with all tokens pre-defined.

Avatar

<Avatar user={user} size={32} />
<Avatar user={null} size={48} />   {/* shows "?" */}

ThemeToggle

Sun/moon icon button. Cycles through: system, light, dark. Uses useTheme() internally.

import { ThemeToggle } from '@freeappstore/sdk/ui'

<ThemeToggle />

No props required. The toggle reads and writes to localStorage('fas:theme') and applies data-theme on <html>.

FasShell

Full zero-config wrapper. Sticky topbar with logo, app name, and profile menu (or sign-in button). Main content area. "Part of FreeAppStore" footer. Optional auth gate.

import { initApp } from '@freeappstore/sdk'
import { FasShell } from '@freeappstore/sdk/ui'

const fas = initApp({ appId: 'my-app' })

function App() {
  return (
    <FasShell app={fas} appName="My App" requireAuth>
      <MyAppContent />
    </FasShell>
  )
}
PropTypeDefaultDescription
appFreeAppStore-SDK instance from initApp()
childrenReactNode-App content
appNamestring?-Name shown in the topbar
requireAuthbooleanfalseShow sign-in gate when not authenticated
showThemeTogglebooleantrueShow theme toggle in topbar/menu

Hooks

useAuth(app)

Auth state and actions. Handles auth.init() automatically.

import { useAuth } from '@freeappstore/sdk/hooks'

const { user, loading, signIn, signOut, deleteAccount } = useAuth(fas)
ReturnTypeDescription
userUser | nullCurrent signed-in user
loadingbooleanTrue during initial auth check
signIn() => voidRedirect to GitHub OAuth
signOut() => voidClear session
deleteAccount() => Promise<void>Wipe KV data + sign out

useTheme()

Zero-provider theme hook. Multiple components share state via DOM + localStorage. No React Context needed.

import { useTheme } from '@freeappstore/sdk/hooks'

const { theme, preference, setPreference } = useTheme()
ReturnTypeDescription
theme'light' | 'dark'Resolved theme (accounts for system preference)
preference'light' | 'dark' | 'system'User's stored preference
setPreference(pref) => voidUpdate preference, applies immediately

Stores preference in localStorage('fas:theme'). Applies data-theme and .dark class on <html>. Listens to prefers-color-scheme media query when preference is 'system'.

Patterns

Auth gate

Require sign-in before showing app content:

<FasShell app={fas} appName="My App" requireAuth>
  <MyApp />
</FasShell>

Custom topbar

Build your own header using individual components:

import { useAuth, useTheme } from '@freeappstore/sdk/hooks'
import { Avatar, ThemeToggle, SignInButton } from '@freeappstore/sdk/ui'

function MyHeader() {
  const { user, signOut } = useAuth(fas)
  return (
    <header>
      <h1>My App</h1>
      <ThemeToggle />
      {user ? <Avatar user={user} /> : <SignInButton app={fas} />}
    </header>
  )
}

Settings page

Add a profile/settings route:

import { ProfilePage } from '@freeappstore/sdk/ui'

<Route path="/settings" element={<ProfilePage app={fas} />} />

Dark mode in 2 lines

Add dark mode support to any app:

import { ThemeToggle } from '@freeappstore/sdk/ui'

// Drop this anywhere — it handles everything:
<ThemeToggle />

The toggle reads/writes localStorage and applies data-theme on <html>. Your CSS just needs to define variables for :root[data-theme='dark'] or html.dark.
