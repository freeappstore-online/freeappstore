Documentation

Everything you need to build a free app. One SDK, one import, full platform access.

Quick Start

# Install the SDK
npm i @freeappstore/sdk

# Initialize in your app
import { initApp } from '@freeappstore/sdk'
const fas = initApp({ appId: 'my-app' })

Your app deploys to my-app.freeappstore.online on push to main.

React Hooks

Hooks give you full control over your UI. Import from @freeappstore/sdk/hooks.

useAuth

Auth state and actions. The primary way to interact with platform identity.

import { useAuth } from '@freeappstore/sdk/hooks'

function App() {
  const { user, loading, signIn, signOut, deleteAccount, hasRole } = useAuth(fas)
  // hasRole('moderator') — async, checks if current user has the role

  if (loading) return <p>Loading...</p>
  if (!user) return <button onClick={signIn}>Sign in with GitHub</button>
  return <p>Welcome, {user.login}!</p>
}

useTheme

Zero-provider theme hook. Reads/writes localStorage, applies data-theme on <html>.

import { useTheme } from '@freeappstore/sdk/hooks'

function ThemeButton() {
  const { theme, preference, setPreference } = useTheme()

  return <button onClick={() => setPreference(theme === 'dark' ? 'light' : 'dark')}>
    {theme === 'dark' ? 'Light mode' : 'Dark mode'}
  </button>
}

Theme values: 'light' | 'dark' (resolved). Preference values: 'light' | 'dark' | 'system'.

Drop-in React components. Import from @freeappstore/sdk/ui. Components use CSS custom properties (--ink, --accent, etc.) so they blend into your app's theme.

FasShell (zero-config)

Full wrapper: auth gate, topbar, theme toggle, footer. One line to get a complete app shell.

import { initApp } from '@freeappstore/sdk'
import { FasShell } from '@freeappstore/sdk/ui'

const fas = initApp({ appId: 'my-app' })

export default function App() {
  return (
    <FasShell app={fas} appName="My App">
      <MyAppContent />
    </FasShell>
  )
}
PropTypeDescription
appFreeAppStoreSDK instance from initApp()

appNamestring?Name shown in the topbar
requireAuthboolean?Show sign-in gate (default: false)
showThemeToggleboolean?Show theme toggle (default: true)

Auth

GitHub OAuth via redirect. Session persists in localStorage. Shared across all FreeAppStore apps (SSO).

await fas.auth.init()                  // Initialize, capture callback
fas.auth.onChange(user => {})          // Listen for sign-in/out
fas.auth.signIn()                      // Redirect to GitHub OAuth
fas.auth.signIn('google')              // Redirect to Google OAuth
fas.auth.signIn('apple')               // Redirect to Apple OAuth
await fas.auth.signInWithEmail(email)  // Send magic-link email
fas.auth.signOut()                     // Clear session
fas.auth.user                          // Current user (or null)
fas.auth.token                         // Session token (for API calls)
await fas.auth.setDateOfBirth('2000-01-15')  // Set DOB (set-once, age >= 13)

Per-user KV

Per-user, per-app key-value store. Each user's data is isolated.

await fas.kv.set('profile', { name: 'Alice', theme: 'dark' })
const profile = await fas.kv.get('profile')
const keys = await fas.kv.list({ prefix: 'note:' })
const items = await fas.kv.getMany(keys)  // fetch multiple keys at once
await fas.kv.delete('profile')

Collections (Firestore-style)

Document-based storage with queries. Shared across users (access-controlled server-side).

const todos = fas.collections.collection('todos')

// Create
await todos.create({ title: 'Buy milk', done: false })

// Query
const { documents, total } = await todos.query({ orderBy: 'title', order: 'desc' })

// Get single document
const doc = await todos.get('doc-id')

// Update (owner only)
await todos.update(id, { done: true })

// Delete (owner only)
await todos.delete(id)

Counters

Cross-user atomic counters for votes, views, leaderboards.

await fas.counters.increment('page-views')       // +1
await fas.counters.increment('likes', -1)          // decrement
const count = await fas.counters.get('page-views') // read (no auth)
const all = await fas.counters.list()               // all counters

Rooms (Real-time WebSocket)

WebSocket rooms for chat, presence, multiplayer. Backed by Cloudflare Durable Objects.

const room = fas.rooms.join('lobby')

room.send({ type: 'chat', text: 'hello' })
room.onMessage(msg => console.log(msg))
room.onPeers(peers => console.log(peers.length, 'online'))
room.close()

Proxy (Secret-injecting API Proxy)

Call third-party APIs without exposing keys to the client.

const response = await fas.proxy.fetch('/openai/chat/completions', {
  method: 'POST',
  body: JSON.stringify({ model: 'gpt-4', messages: [...] }),
})

Roles (App-level RBAC)

Per-app role management. Every app gets a set of default roles out of the box. Roles are scoped to your app and enforced server-side.

Default roles (available without configuration):

  • owner — auto-assigned to the app creator
  • member — default role for signed-in users
  • moderator — can manage content and users
  • editor — can create and edit content
  • viewer — read-only access
// Assign a role
await fas.roles.assign(userId, 'moderator')

// Revoke a role
await fas.roles.revoke(userId, 'moderator')

// Check if the current user has a role (async — hits the API)
const isMod = await fas.roles.check('moderator') // true | false

// Get the current user's roles (async — hits the API)
const roles = await fas.roles.myRoles() // ['owner', 'moderator']

Custom roles are also supported — pass any string as the role name. Custom roles work identically to the built-in defaults.

// Custom roles
await fas.roles.assign(userId, 'beta-tester')
const isBeta = await fas.roles.check('beta-tester')

AI Agent Integration

Two ways for AI agents to access the platform:

MCP Server (recommended)

Connect your AI agent directly to FreeAppStore tools via MCP:

// Add to ~/.claude.json (Claude Code) or your MCP client config:
{
  "mcpServers": {
    "freeappstore": {
      "command": "npx",
      "args": ["mcp-remote", "https://mcp.freeappstore.online/mcp"]
    }
  }
}

Tools: deploy_status, app_info, sdk_reference, platform_guide, list_apps. Full MCP docs →

SKILLS.md (text-based)

For agents without MCP support, point them at the machine-readable guide:

# Add to your CLAUDE.md:
See https://freeappstore.online/skills.md for platform skills.
