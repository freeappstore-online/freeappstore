Privacy Rules

Non-negotiable. Every app on FreeAppStore must follow these:

Zero tracking

No analytics, no telemetry, no usage metrics. Zero network requests for tracking purposes.

No cookies

No cookies of any kind — not for sessions, not for preferences, not for anything.

localStorage only

All user data stays on-device in localStorage. No server-side storage for standalone apps. No external scripts (CDN fonts are OK).

Compliance Checks

These checks run automatically on every push. Your app must pass all of them.

Build & Code

  • TypeScript strict mode — no any types
  • Clean build with zero warnings
  • Core JS bundle <100KB gzipped
  • MIT license present in repo

Design

  • Uses Manrope + Fraunces fonts
  • Uses shared CSS custom properties for neutrals
  • Single app-specific accent color
  • Sidebar (desktop) + dock (mobile) shell pattern
  • Dark mode via system preference
  • Respects safe area insets (notched phones)

PWA

  • manifest.json with name, icons, theme color
  • apple-mobile-web-app-capable meta tag
  • Service worker registered
  • Works when added to home screen
  • Works offline after first load

Privacy

  • Zero tracking/analytics network requests
  • No cookies
  • No external scripts (CDN fonts OK)
  • All user data in localStorage only

Performance

  • First load <2s on 3G
  • No layout shift after load
  • Works on mobile (320px+) and desktop (1024px+)

Templates

Standalone

Pure client-side app, no backend. All data in localStorage. Works fully offline.

github.com/freeappstore-online/template-standalone

Connected

Apps with auth, per-user storage, realtime rooms, or API proxy. Start from standalone template, add @freeappstore/sdk.

npm i @freeappstore/sdk
