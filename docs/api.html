API Reference

Public REST endpoints on api.freeappstore.online. All responses are JSON unless noted otherwise.

Base URL

https://api.freeappstore.online

Authentication

Endpoints marked Auth required expect a Bearer token in the Authorization header. Obtain a token by completing the GitHub OAuth flow (see below).

Authorization: Bearer <token>

Auth

GET /v1/auth/github/start

Start the GitHub OAuth sign-in flow. Redirects the browser to GitHub. No auth required.

Query paramRequiredDescription
app_idYesThe app requesting sign-in (use store for the storefront itself)
return_toYesURL to redirect back to after authentication completes
 
# Open in a browser (redirect flow)
curl -v "https://api.freeappstore.online/v1/auth/github/start?app_id=store&return_to=https://freeappstore.online/"

GET /v1/auth/me

Return the currently authenticated user. Auth required.

curl -s https://api.freeappstore.online/v1/auth/me \
  -H "Authorization: Bearer $FAS_TOKEN"

Response:

{
  "id": "github|12345",
  "login": "octocat",
  "avatar": "https://avatars.githubusercontent.com/u/12345",
  "roles": ["user"]
}

Quality

GET /v1/apps/:id/quality

VCQA code health report for an app. Returns the full scan results as JSON. No auth required.

Path paramDescription
:idApp identifier (e.g. timer, notes)
 
curl -s https://api.freeappstore.online/v1/apps/timer/quality

Response:

{
  "appId": "timer",
  "score": 82,
  "grade": "A",
  "checks": [
    { "name": "has-manifest", "pass": true },
    { "name": "responsive", "pass": true },
    ...
  ]
}

GET /v1/apps/:id/quality/badge

SVG badge showing the app's code quality grade. Suitable for embedding in README files. No auth required.

# Embed in markdown:
# ![quality](https://api.freeappstore.online/v1/apps/timer/quality/badge)

curl -s https://api.freeappstore.online/v1/apps/timer/quality/badge
# Returns: image/svg+xml

Audit

GET /v1/audit

Compliance audit summary across all apps in a store. No auth required.

Query paramRequiredDescription
storeYesStore to audit (e.g. apps)
appNoFilter to a single app ID
 
# All apps
curl -s "https://api.freeappstore.online/v1/audit?store=apps"

# Single app
curl -s "https://api.freeappstore.online/v1/audit?app=timer"

Response:

{
  "summary": [
    {
      "appId": "timer",
      "pass": 12,
      "warn": 1,
      "fail": 0
    }
  ]
}

Publish

POST /v1/publish

Provision a new app on the platform. Creates the GitHub repo, R2 hosting, custom subdomain, DNS record, and registry entry. Auth required.

curl -s -X POST https://api.freeappstore.online/v1/publish \
  -H "Authorization: Bearer $FAS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My App",
    "id": "my-app",
    "description": "A useful productivity tool",
    "category": "productivity"
  }'

Response:

{
  "ok": true,
  "appUrl": "https://my-app.freeappstore.online",
  "repo": "freeappstore-online/my-app"
}

This endpoint is idempotent -- calling it again with the same id is safe and re-runs any incomplete provisioning steps.

Analytics

GET /v1/analytics.js

Cloudflare Web Analytics loader script. Injected automatically by the platform for published apps. No auth required.

Query paramRequiredDescription
appYesApp identifier
 
# Returns: application/javascript
curl -s "https://api.freeappstore.online/v1/analytics.js?app=timer"

In HTML:

<script defer src="https://api.freeappstore.online/v1/analytics.js?app=timer"></script>

Apps published through the platform get this script injected automatically. You only need to add it manually if you are self-hosting or customizing your deploy.
